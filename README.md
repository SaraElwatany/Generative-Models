## Overview

This branch contains **ResNet-18 classifiers** trained on each dataset used in the project. These classifiers are not generative models — they serve exclusively as **evaluation tools** for computing the **Inception Score (IS)** of images generated by the VAE and DDPM models.

Training dataset-specific classifiers (rather than using a generic ImageNet-pretrained network) ensures the Inception Score faithfully reflects class-level quality and diversity within each dataset's own distribution.

## Purpose

The Inception Score requires a classifier that can assign **class probabilities** $p(y|x)$ to generated images. These probabilities are then used to evaluate two properties of the generative model:

- **Confidence** — are generated images clearly recognisable as a specific class?
- **Diversity** — are generated images spread across all possible classes, or does the model collapse to a few?

$$\text{IS} = \exp\left(\mathbb{E}_x\, D_{KL}(p(y|x) \| p(y))\right)$$

**Higher IS → images are both high-quality and class-diverse.**

Using dataset-specific classifiers rather than a generic pretrained network ensures the score reflects the actual class structure of each dataset (e.g., 102 flower categories rather than ImageNet's 1000 classes).

## How It Works

```
Generated Images (VAE / DDPM)
            │
            ▼
   ResNet-18 Classifier
   (trained on same dataset)
            │
            ▼
  Class Probabilities p(y|x)
            │
       ┌────┴────┐
       ▼         ▼
  Confidence  Diversity
  p(y|x) is  p(y|x) is
  peaky       spread
       │         │
       └────┬────┘
            ▼
     Inception Score (IS)
```

1. Generated images from the VAE or DDPM are fed into the trained classifier
2. The classifier outputs a probability distribution over classes $p(y|x)$ for each image
3. The marginal distribution $p(y) = \mathbb{E}_x[p(y|x)]$ is computed across all generated images
4. IS is computed as the exponentiated KL divergence between $p(y|x)$ and $p(y)$

## Datasets

| Dataset | Classes | Resolution | Training Samples |
|---|---|---|---|
| **MNIST** | 10 (digits 0–9) | Resized to 32×32x3 grayscale | ~60,000 |
| **CIFAR-10** | 10 (vehicles, animals) | 32×32 RGB | 50,000 |

## How to Use

### Option 1 — Run on Kaggle (Recommended)

[![Open in Kaggle](https://kaggle.com/static/images/open-in-kaggle.svg)](https://www.kaggle.com/code/saraaymanelwatany/classifiers-for-inceptionscore)

Steps:
1. Open the notebook on Kaggle.
2. Enable GPU: `Settings → Accelerator → GPU T4 x2` (or P100).
3. Select your dataset: set `DATASET_INDEX  = INDEX` where INDEX: <kbd>0</kbd> → MNIST · <kbd>1</kbd> → CIFAR-10 · <kbd>2</kbd> → Oxford Flowers
4. Run all cells — the notebook will load the dataset, train ResNet-18, and save the best checkpoint.

---

### Option 2 — Run Locally

#### 1. Clone the repository and switch to this branch

```bash
git clone https://github.com/SaraElwatany/Generative-Models.git
cd Generative-Models
git checkout feat/classifiers
```

#### 2. Install Git LFS (for checkpoint files)

```bash
git lfs install
git lfs pull
```

#### 3. Install dependencies

```bash
pip install torch torchvision numpy matplotlib scikit-learn tqdm
```

#### 4. Train a classifier

Open `classifiers-for-inceptionscore.ipynb` and set:

```python
DATASET_INDEX  = 0    # 0: MNIST | 1: CIFAR-10 | 2: Oxford Flowers
```

Run all cells — the best model will be saved to `checkpoints/`.

#### 5. Load a checkpoint for IS evaluation

```python
import torch
from torchvision.models import resnet18

model = resnet18()
model.load_state_dict(torch.load("checkpoints/mnist_best_classifier.pt"))
model.eval()
```
